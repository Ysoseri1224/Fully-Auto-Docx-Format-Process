# WriteMaster 可视化模板提取工作台设计稿

## 目标

把当前偏“专用脚本链”的模板处理过程，提升为一个可视化的模板提取与规则确认工作台。

核心目标不是把 DOCX 转成 Markdown 或 JSON，而是把模板文档中的：

- 段落样式
- 编号层级
- 标题体系
- 图题表题
- 代码块
- 提示说明块
- 表格对象
- 页面与版式线索

提取为一套**可确认、可保存、可复用**的模板配置 profile。

---

## 核心定位

这个工作台不是 Word 编辑器，也不是通用排版软件。

它的定位是：

1. 读取母版 DOCX
2. 解析为结构化中间模型
3. 允许用户在预览中标注语义角色
4. 生成临时样式与规则配置
5. 用这些规则去作用于 Markdown / DOCX 输入
6. 再通过预览回看结果并继续微调

---

## 总体流程

### 阶段 A：模板提取

1. 上传母版 DOCX
2. 解析成中间结构
3. 在界面中查看原文预览 + 结构 JSON + 候选样式
4. 用户右击某段/某标题/某图题/某表格块
5. 标记为某个语义角色，或存为临时样式
6. 生成模板 profile

### 阶段 B：规则应用

1. 上传 Markdown 或目标 DOCX
2. 选择一个已有模板配置
3. 生成结果预览
4. 对照修订样式映射或临时样式定义
5. 导出最终 DOCX

## 页面结构

建议拆成四个区域。

### 1. 左侧边栏

职责：

- 新建模板提取任务
- 新建应用任务
- 查看最近模板配置
- 查看最近文档
- 切换“模板提取 / 应用结果”模式

建议条目：

- 新建模板提取
- 新建结果预览
- 模板配置
- 最近文件
- 导入配置
- 导出配置

### 2. 中间主预览区

职责：

- 展示母版 DOCX 的近似页面预览
- 允许点击段落、标题、图题、表格块
- 允许右击并调用语义标注菜单

预览要求：

- 第一阶段不追求和 Word 完全一致
- 优先保证块级结构正确
- 支持：
  - 标题
  - 正文
  - 列表
  - 代码块
  - 图题/表题
  - 表格
  - 提示说明块

### 3. 右侧信息区

上半区：结构 JSON / 中间 AST

- 当前选中节点信息
- 段落文本
- 原始样式来源
- 编号信息
- 字体字号
- 缩进/对齐/行距
- 所属页与块类型

下半区：临时样式 / 规则配置

- 已定义临时样式列表
- 当前节点可归入哪个临时样式
- 当前模板 profile 的核心字段

### 4. 下方结果区或切换页

职责：

- 展示 Markdown 或 DOCX 应用当前模板配置后的结果预览
- 用于“模板提取后回看”

这一块建议做成 Tab，而不是首版就做上下双屏。

Tab 建议：

- 原始模板视图
- 结构 JSON
- 应用结果预览
- Profile 配置

## 中间数据模型

建议不要叫“解析成 docx”，而是定义自己的中间结构。

例如：

```json
{
  "document": {
    "pages": [],
    "blocks": [
      {
        "id": "p-0012",
        "type": "paragraph",
        "role": "unknown",
        "text": "10.1 模拟基础理论与方法",
        "styleSource": {
          "styleId": "Heading2",
          "styleName": "Heading 2",
          "directPPr": {},
          "directRPr": {}
        },
        "numbering": {
          "numId": null,
          "ilvl": null,
          "displayText": null
        },
        "format": {
          "fontFamily": "黑体",
          "fontSizePt": 16,
          "bold": true,
          "italic": false,
          "align": "left",
          "firstLineIndentChars": 0,
          "lineSpacing": 1.5
        },
        "location": {
          "pageIndex": 12,
          "order": 4
        }
      }
    ]
  }
}
```

重点是：

- `type` 表示块类型
- `role` 表示用户确认后的语义角色
- `styleSource` 表示原始来源
- `format` 表示可聚类的视觉信息

## 临时样式系统

### 定义

“临时样式”不是 Word 的 styleId，而是工作台内部的语义样式候选。

例如：

- `tmp_body_01`
- `tmp_heading_02`
- `tmp_caption_01`
- `tmp_note_01`

### 生命周期

1. 系统自动提取并聚类
2. 用户确认语义
3. 用户可合并/重命名
4. 最终映射到模板 profile 字段

### 用户可执行操作

- 存为临时样式
- 并入已有临时样式
- 标记为正文候选
- 标记为一级标题候选
- 标记为二级标题候选
- 标记为三级标题候选
- 标记为图题
- 标记为表题
- 标记为代码块
- 标记为提示说明
- 标记为忽略

## 右击菜单建议

当用户右击一个段落/块时，显示：

- 设为正文
- 设为一级标题
- 设为二级标题
- 设为三级标题
- 设为图题
- 设为表题
- 设为代码块
- 设为提示说明
- 设为列表一级
- 设为列表二级
- 存为新的临时样式
- 合并到已有临时样式
- 忽略该块

- 设为表格模板
- 设为对照表
- 设为普通数据表
- 将首行设为表头
- 标记单元格对齐模式

## 第一阶段最小可实现范围

首版建议只做这些：

### 做

- 母版 DOCX 上传
- 结构提取
- 原文近似预览
- 结构 JSON 查看
- 右击段落标记语义
- 临时样式面板
- 生成 profile JSON
- “应用现有模板配置”下拉栏
- 内置当前项目模板配置

### 不做

- 批注编辑
- 交叉引用修复
- 目录域重建
- 公式对象重排
- 页眉页脚复杂联动
- 分节/多栏复杂布局
- 真正的 Word 所见即所得编辑

---

## 主要风险

### 1. 没有真实 style，只能靠视觉聚类

很多模板没有自定义样式，只有直接格式。

风险：

- 同一种视觉正文会被提成多个候选
- 需要聚类与合并

### 2. 预览块与 OOXML 节点映射难

用户点到的预览对象必须精确映射回 `w:p` / `w:tbl`。

### 3. 结果视图和模板视图不是同一数据源

一个是模板抽取视图，一个是规则应用视图，必须分模式处理。

### 4. 范围容易失控

若把目录、批注、公式、交叉引用全部一起做，会迅速变成 Word 替代品项目。

---

## UI 原则

- 功能描述优先，不写宣传口吻
- 先让用户“看懂自己在做什么”
- 不一次展示全部高级字段
- 默认显示：
  - 结构
  - 候选
  - 当前映射
- 高级 OOXML 字段折叠显示
